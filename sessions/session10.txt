BUG-1: census.py missing _ingest_extra() — FIXED
---------------------------------------------------
Problem: run() called TRUNCATE on core_census_lsoa then only populated 4 of 5 domains.
  The 5th domain (extra: health, cars, econ, degree, born_abroad) had no ingest function.
  Next pipeline run would wipe the extra columns populated by migration 007.

Fix (two parts):
1. Removed TRUNCATE from run() — now relies on ON CONFLICT DO UPDATE (pure upsert).
   Extra columns survive even when their source CSVs are unavailable.
2. Added _ingest_extra(cur, census_dir) function (~160 lines) handling 5 Census datasets:
   - TS037 (General health → pct_good_health): "Very good" + "Good" / Total
   - TS045 (Car/van availability → pct_no_car): "No cars or vans" / Total
   - TS066 (Economic activity → pct_economically_active): "Economically active (excl students)" / Total
   - TS067 (Qualifications → pct_degree): "Level 4 or above" / Total
   - TS004 (Country of birth → pct_born_abroad): (Total - UK) / Total
Each dataset processes independently; missing CSVs skipped with warning.
Source CSVs (LSOA level) need download from ONS/Nomis — only ward-level available locally.
DB data safe: all 33,755 rows have all 5 columns populated from migration 007.
