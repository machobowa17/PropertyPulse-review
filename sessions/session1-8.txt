--- PERFORMANCE FIXES (Step 3g) ---

1. uk_median query: subquery SELECT MAX(date_of_transfer) → CURRENT_DATE - INTERVAL '13 months'
   PostgreSQL planner can't estimate selectivity for subquery-based date filters.
   Using CURRENT_DATE constant allows bitmap index scan. Result: 50s → 7s

2. Choropleth avg_price: Added AND lsoa_code = ANY(:codes) to inner query
   Was scanning full 28.9M row table; now scoped to requested LSOAs only.

3. Choropleth price_per_sqft: Same — added lsoa_code = ANY(:codes)

4. ppsm_local + ppsm_parent: Added AND date_of_transfer >= CURRENT_DATE - INTERVAL '25 months'
   No date scope was causing GROUP BY to scan 2M+ rows.

Post-fix result: 137/138 passed, 1 expected (M1 semi-detached no recent sales)
